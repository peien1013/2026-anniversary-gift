# 2026週年慶紀念品－吊飾｜登記購買系統

一套「沉浸互動風、分步填寫」的紀念品登記購買系統。
長輩友善：字大、按鈕大、一次只填一步、白話錯誤提示。

```
前台 GitHub Pages (index.html)
        │  JSONP
        ▼
後端 Apps Script Web App (Code.gs)
        │
        ▼
資料庫 Google 試算表（系統設定 / 商品設定 / 登記資料 / 統計）
```

---

## 一、檔案結構

```
2026週年慶紀念品/
├─ index.html          ← 前台（CSS/JS 已整合，單檔即可部署）
├─ README.md           ← 本說明
└─ apps-script/
   └─ Code.gs          ← Google Apps Script 後端
```

---

## 二、建立後端（Google 試算表 + Apps Script）

1. 到 Google 雲端硬碟 → 新建一個**空白試算表**（名稱自取，例如「2026週年慶紀念品後台」）。
2. 上方選單 **擴充功能 → Apps Script**。
3. 把 `apps-script/Code.gs` 全部內容貼進去，**覆蓋**原本的 `Code.gs`，按存檔。
4. 回到試算表，**重新整理頁面**，上方會多出選單 **「紀念品系統」**。
5. 點 **紀念品系統 → 初始化（只建立缺少的工作表）**（第一次會要求授權，按允許）。
   - 會自動建立四張工作表：**系統設定、商品設定、登記資料、統計**，並填入示範資料。
   - ⚠️ 這是**非破壞式**：已存在的「系統設定／商品設定」**不會被覆蓋**（你填的內容會保留）。要改商品結構或清空登記，請用下面的「重建 登記資料表」。

### 部署成 Web App

6. Apps Script 編輯器右上 **部署 → 新增部署**。
7. 類型選 **網頁應用程式**：
   - 說明：隨意（例如 v1）
   - **執行身分：我**
   - **誰可以存取：任何人**
8. 按 **部署**，複製產生的 **網頁應用程式 URL**（結尾是 `/exec`）。

> 之後若**修改 `Code.gs`**：要回到 **部署 → 管理部署作業 → ✏️編輯 → 版本選「新版本」→ 部署**。
> 這樣**網址不會變**，前台才不用重設。（按「新增部署」會產生新網址。）

## 三、設定前台

打開 `index.html`，找到最上方這一行，把網址換成你剛剛複製的 `/exec`：

```js
var GOOGLE_SCRIPT_URL = 'https://script.google.com/macros/s/請填入你的部署ID/exec';
```

## 四、部署前台到 GitHub Pages

1. 建一個 GitHub repo（例如 `memorabilia-2026`）。
2. 上傳 `index.html`（放在根目錄）。
3. repo → **Settings → Pages** → Source 選 **main 分支 / root** → 儲存。
4. 等一兩分鐘，會給你一個網址（`https://你的帳號.github.io/memorabilia-2026/`），這就是發給大家的登記網址。

> 改過 `index.html` 後，使用者要 **Ctrl+Shift+R（或無痕視窗）** 才會看到最新版（GitHub 與瀏覽器都有快取）。

## 五、Google 試算表欄位說明

### 工作表 1｜系統設定（key-value，可直接改）

| 設定項目 | 範例 | 說明 |
|---|---|---|
| 系統標題 | 2026週年慶紀念品－吊飾 | 前台頁首大標 |
| 活動名稱 | 2026週年慶紀念品－吊飾 | |
| 活動日期 | 2026週年慶當日 | 自由填寫 |
| 是否開放登記 | 開放 | 填「開放」才可登記；「關閉」前台只顯示公告 |
| 登記截止時間 | （空）或 2026/09/30 23:59 | 留空＝不限制 |
| 公告文字 | 歡迎登記… | 公告／關閉時顯示 |
| 成功頁提示文字 | 取件或寄送時間將另行通知。 | |
| 郵寄費 | 100 | 選郵寄時加收 |
| 週年慶當日取件是否開放 | 開放 | 是／否 |
| 高雄道場提前取件是否開放 | 開放 | 是／否 |
| 郵寄是否開放 | 開放 | 是／否 |

### 工作表 2｜商品設定（一列一個商品）

| 欄位 | 說明 |
|---|---|
| 商品ID | 唯一代號（例 P01），請勿與其他列重複 |
| 商品名稱 | |
| 商品說明 | |
| 單價 | 數字 |
| 圖片連結1 / 圖片連結2 | 貼 Google Drive **分享連結**即可（可放正面／背面兩張） |
| 是否顯示 | 是／否（否＝前台不顯示） |
| 顯示順序 | 數字小的排前面 |
| 是否可購買 | 否＝前台顯示但不能加數量 |
| 數量上限 | 單筆每項最多可買幾個 |
| 備註 | 自由 |

> 本系統為**單一商品**設計，「登記資料」只記錄一項商品（商品名稱／單價／數量）。
> 前台仍會自動讀取商品設定第一個「是否顯示＝是」的商品。

### 工作表 3｜登記資料（每筆登記一列，自動寫入）

欄位順序（單一商品設計）：建立時間、姓名、**商品名稱、單價、數量**、商品總金額、取貨方式、郵寄費、總金額、收件人、收件人電話、配送方式、超商名稱、門市名稱、門市地址、郵遞區號、完整收件地址、付款狀態、取件／寄送狀態、管理備註、是否取消、**備註**、**代領家人（最後一欄）**。

> 「**備註**」是登記人在前台選填的留言；「**管理備註**」給你自己內部註記用，兩者分開。

- **付款狀態**下拉：未付款／已付款／不需付款／已退款
- **取件／寄送狀態**下拉：未處理／待取件／已取件／待寄出／已寄出／已完成
- 管理者可手動修改付款／寄送狀態、加管理備註。
- **金額自動重算**：你手動改某列的 **數量／單價／郵寄費**，該列的 **商品總金額、總金額** 會自動跟著算；「統計」頁的加總也會自動更新。若一次改很多列、想保險重算一次，按選單 **紀念品系統 → 重算所有金額**。
- **重複姓名標紅底**：選單 **紀念品系統 → 標記重複姓名（紅底）** 套用一次後，只要「姓名」欄有重複，那些格子會**自動變紅底**（含日後新登記），方便你一眼看到去處理。

#### 代領家人（會員免費紀念品）

- 最後一欄「**代領家人**」**由你手動填**：寫上這筆要「順便代領的會員姓名」，多個用 **、** 分隔（例：`陳大嫂、陳小弟`）。
- 對方**查詢、且取貨方式為「郵寄」**時，才會看到：
  - 🎁 會員紀念品（代領家人）＋ 人數
  - 📦 郵寄內容共 ＝ **代領人數 ＋ 額外購買數量**
- **非郵寄**（當日／高雄道場取件）不顯示代領；當日現場領即可。
- 沒填就不顯示，完全由你決定。系統**不會自動加免費數量**。

> ⚠️ 欄位是用**位置**讀寫的。要新增欄位請**加在最後面**；不要插在中間或調動順序。

### 工作表 4｜統計（公式自動更新）

登記總筆數、各商品總數量／總金額、商品總收入、郵寄件數、郵寄費總額、應收總金額、已付款／未付款金額、各取貨方式人數、超商／宅配件數、已取件／已寄出／已完成數。
改了登記資料就會自動重算；若公式跑掉，選單 **紀念品系統 → 重建 統計頁**。

> 「**商品總數量**」＝所有人額外購買的總數（你要統計的「額外登記了幾個」就看這格）。

### 工作表 5｜郵寄清單（按選單一鍵產生）

選單 **紀念品系統 → 產生 郵寄清單**，會把所有「取貨方式＝郵寄、未取消」的人整理成一張表：

| 欄位 | 說明 |
|---|---|
| 登記時間 / 姓名 / 收件人 / 收件人電話 | 基本聯絡 |
| 配送方式 | 超商店到店 / 宅配地址 |
| 寄送地址 / 門市 | 宅配＝郵遞區號＋地址；超商＝超商＋門市（＋地址） |
| 額外購買 | 這筆買的數量 |
| 代領家人 | 你在「登記資料」手填的代領名單 |
| **合計件數** | 額外購買 ＋ 代領人數＝這箱要放幾個 |
| 付款狀態 / 寄送狀態 | 方便核對 |

最後一列有 **合計**（額外購買總數、合計件數總數）。每次有新登記就重新「產生 郵寄清單」即可（會覆蓋重建）。

## 六、Google Drive 圖片分享連結用法

1. 圖片上傳到 Google 雲端硬碟。
2. 右鍵 → **共用** → 一般存取權改成 **「知道連結的任何人」→ 檢視者**。
3. 點 **複製連結**，貼到「商品設定」的 **圖片連結1 / 圖片連結2** 欄位。

後端會自動把分享連結轉成可顯示格式（`drive.google.com/thumbnail?id=...`），
所以你只要貼分享連結即可；**換圖只要換連結，前台自動更新**。

## 七、安全設計（已內建）

- 後端用 **LockService** 防併發、**批次寫入**整列。
- 所有金額由**後端依商品設定重新計算**，不信任前端傳來的單價／總額。
- 取貨方式、必填欄位**前後端雙重驗證**。
- 文字若以 `= + - @` 開頭，寫入前自動加單引號，**防公式注入**。
- 試算表 ID、設定不會暴露在前台；前台只透過 Apps Script 取得必要資料。
- 前台送出有**防重複點擊**（送出中鎖按鈕、成功後清除 localStorage）。

## 九、常見問題

- **前台一直「載入失敗」**：多半是 `GOOGLE_SCRIPT_URL` 沒填對，或部署存取權不是「任何人」。
- **圖片不顯示**：確認 Drive 檔案共用權限是「知道連結的任何人」。
- **改了 Code.gs 沒生效**：要「管理部署作業 → 編輯 → 新版本」重新部署。
- **改了 index.html 沒生效**：Ctrl+Shift+R 強制重新整理或用無痕視窗。
- **「登記資料」欄位跑掉／對不齊**：選單 **紀念品系統 → 重建 登記資料表（會清空登記）** → 按「是」即可重建成正確欄位（會清掉現有登記列，測試階段沒差）。
